# 提示词管理与顶栏入口设计

## 1. 背景

当前 `AI 图片` 节点只有一个自由输入的提示词文本框。用户如果经常需要复用人设、镜头、风格、构图等模板，只能手动重写或从外部复制，效率低且容易不一致。

项目现有能力已经表明，这类模板更适合做成“用户级长期配置”而不是“项目级数据”：

- `settingsStore` 已经承载 API Key、下载路径、自定义供应商等跨项目配置。
- `ImageEditNode` 具备成熟的 `promptDraft + promptDraftRef + updateNodeData` 同步链路，适合接入模板应用行为。
- 顶栏已有全局入口区，适合承接一个新的全局管理按钮。

经过本轮讨论，需求已经确认了四个关键交互结论：

1. 管理入口放在顶栏设置按钮旁。
2. 点击入口后直接打开独立的“提示词管理”弹窗。
3. `AI 图片` 节点顶部快捷区使用“两行换行式”布局，超出后内部滚动。
4. 点击快捷标题时，空文本直接写入；非空文本先插入一个空行，再追加模板内容。

## 2. 方案对比

### 2.1 方案 A：顶栏独立按钮 + 独立提示词管理弹窗

这是本次最终采用的方案。

优点：

- 入口路径最短，最符合“设置按钮旁新增一个入口”的要求。
- 不再给 `SettingsDialog` 增加新的分类和信息密度。
- 与当前项目常见的弹窗交互保持一致，设计和实现都更稳。

代价：

- 顶栏会多一个全局按钮，需要控制图标和 hover 态的辨识度。

### 2.2 方案 B：顶栏独立按钮 + 打开设置页中的提示词分类

优点：

- 复用现有设置容器更多。

缺点：

- 用户仍然会落回设置体系，和“独立管理入口”不完全一致。
- 会继续扩大设置页的职责。

### 2.3 方案 C：顶栏独立按钮 + 抽屉式管理面板

优点：

- 视觉上更轻。

缺点：

- 当前项目主交互语言不是抽屉，而是弹窗。
- 需要处理新的遮罩、层级和滚动规则，复杂度不必要。

## 3. 最终设计

### 3.1 数据模型

提示词模板作为全局用户偏好，存入 `settingsStore` 持久化。

建议数据结构：

```ts
interface PromptTemplate {
  id: string;
  title: string;
  content: string;
}
```

配套约束：

- `title` 不能为空
- `content` 不能为空
- `title.trim()` 后必须唯一
- 新增模板默认追加到末尾
- 编辑不改变顺序
- 删除直接移除

### 3.2 入口与弹窗结构

入口放在 `TitleBar` 右侧按钮区，紧邻设置按钮。点击后直接打开独立的“提示词管理”弹窗。

弹窗主体结构：

- 顶部标题
- “添加提示词”按钮
- 模板列表

列表每行展示：

- 模板标题
- 模板内容摘要
- 编辑按钮
- 删除按钮

交互要求：

- 点击“添加”打开新增弹窗
- 点击“编辑”打开编辑弹窗
- 点击“删除”先弹出二次确认
- 保存或删除成功后立即刷新列表
- 列表为空时显示空状态

### 3.3 添加/编辑弹窗

新增和编辑共用一套表单结构：

- 标题输入框
- 提示词内容多行文本框
- 底部“保存”按钮

校验失败时：

- 弹窗保持打开
- 在对应字段附近显示错误提示
- 不做额外异步 loading 设计

原因是模板读写基于本地持久化，不需要引入远端请求式的复杂反馈。

### 3.4 `AI 图片` 节点快捷区

快捷区放在 `ImageEditNode` 的 `NodeHeader` 下方、提示词文本框上方。

布局规则：

- 仅当存在模板时显示
- 按保存顺序展示
- 允许换行
- 最多显示两行高度
- 超出后在快捷区内部滚动

这样既贴近用户参考图，也能避免模板区吞掉主编辑区空间。

### 3.5 模板应用规则

点击快捷标题后，统一按以下规则应用：

- 当前 prompt 为空：直接写入模板内容
- 当前 prompt 非空：在原文本末尾插入一个空行，再追加模板内容

同步更新目标：

- `promptDraft`
- `promptDraftRef`
- `data.prompt`

不做的能力：

- 不按光标位置插入
- 不做模板自动去重
- 不做模板分类和搜索

## 4. 风险与边界

- 顶栏新增按钮会增加一个全局入口，但这比继续把提示词管理塞进设置页更清晰。
- 追加式模板应用可能带来重复内容，但这是可预期的显式行为，优先保证简单和确定性。
- 当模板很多时，节点快捷区可能变拥挤，因此强制限制为最多两行，并在区域内部滚动。
- 本期模板是全局用户级能力，不处理项目级隔离需求。

## 5. 验证建议

建议至少覆盖以下验证：

1. `settingsStore` 中模板的新增、编辑、删除、空值校验和标题判重。
2. 提示词管理弹窗中的添加、编辑、删除确认、列表刷新和空状态。
3. `ImageEditNode` 中“无模板不显示快捷区”“模板区两行可滚动”“空文本写入”“非空文本空行后追加”。
4. i18n 文案在中英文下均可正常显示。

## 6. 与 OpenSpec 对齐

本设计对应的 OpenSpec change 为：

- `openspec/changes/add-image-prompt-management/`

需要保持一致的关键口径：

- 不再使用“设置页提示词分类”作为入口。
- 不再使用“点击模板后整段替换 prompt”的交互定义。
- 统一采用“顶栏独立入口 + 独立管理弹窗 + 节点顶部快捷区 + 空文本写入/非空文本空行后追加”。
